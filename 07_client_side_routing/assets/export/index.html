1Ô∏è‚É£ BrowserRouter

üí° BrowserRouter is a wrapper for App to allow conditional rendering based on the URL

‚ùì Where does it belong?

Since the App component is imported and mounted inside the index.js file, this is a great place to wrap the component within BrowserRouter

 
import React from "react";
import ReactDOM from "react-dom";
import { BrowserRouter as Router } from "react-router-dom";
import "./index.css";

import App from "./App";

ReactDOM.render(
  <React.StrictMode>
    <Router>
      <App />
    </Router>
  </React.StrictMode>,
  document.getElementById("root")
);

Switch Component

üí° Switch is a wrapper for Routes. Looks through all its child Route components and returns the first one that matches the current URL (like a switch statement in JavaScript)

‚ùì Where does it belong?

Consider where most of the applications components are rendered, this is where they will be nested inside the Switch component.

Typically this is in the App component. Sometimes an extra component will be created designated for just routing.

return (
  <div className={isDarkMode ? "App" : "App light"}>
    <Header 
      isDarkMode={isDarkMode} 
      onToggleDarkMode={onToggleDarkMode} 
    />
    <Switch>
      <Home />
      <About />
      {renderForm()}
      <ProjectList
        projects={projects}
        onEditProject={onEditProject}
        onDeleteProject={onDeleteProject}
        setSelectedPhase={setSelectedPhase}
        setSearchQuery={setSearchQuery}
      />
      <Home />
      <ProjectDetail />
    </Switch>
  </div>
);

Creating routes using the Route Component

üí° Route wraps content that should be visible when the designated route is active.

‚ùì Where does it belong?

Every component nested inside of the Switch component will be individually wrapped inside of a Route component.

üí• Route will be provided a path prop. {.fragment} if the current URL matches the path, the Route will render its children. Otherwise, the Route renders null.

return (
  <div className={isDarkMode ? "App" : "App light"}>
    <Header 
      isDarkMode={isDarkMode} 
      onToggleDarkMode={onToggleDarkMode} 
    />
    <Switch>
      <Route exact path="/">
        <Home />
      </Route>
      <Route path="/projects/new">
        <ProjectForm onAddProject={onAddProject} />
      </Route>
      <Route path="/projects/:id/edit">
        <ProjectEditForm onUpdateProject={onUpdateProject} />
      </Route>
      <Route path="/projects/:id">
        <ProjectDetail />
      </Route>
      <Route path="/projects">
        <ProjectList projects={projects} onDeleteProject={onDeleteProject} />
      </Route>
    </Switch>
  </div>
);

Link Component

üí° Link creates an anchor tag in your application that will navigate using React Router instead of the browser default

We want to use Link for navigation in our application. It will ensure that a page reload does not occur, unlike the use of an anchor tag <a></a>

üí• Link will be provided a to prop which indicates where the link should navigate to (generates the href)

return (
  <header>
    <nav>
      <Link to="/" style={{ borderBottom: "none" }}>
        <h1 className="branding">
          <span className="logo">{"//"}</span>
          Project Showcase
        </h1>
      </Link>
      <div className="navigation">
        <Link className="button" to="/projects">
          All Projects
        </Link>
        <Link className="button" to="/projects/new">
          Add Project
        </Link>
        ...
      </div>
    </nav>
  </header>
);
